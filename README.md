# HomePodMonitor

HomePodMonitor 是一个 macOS 菜单栏应用，用来监控当前音频输出设备，并在输出不是目标设备时，将系统输出切回用户选定的目标设备。

项目当前基于 SwiftUI 构建，运行形态是 MenuBarExtra 窗口，不显示常规 Dock 应用窗口。

## 功能

- 菜单栏常驻显示当前是否已经连接到目标输出设备
- 读取系统声音菜单中的输出列表并展示在应用菜单中
- 用户可在列表中把任意输出设为默认输出
- 设置默认输出时会立即尝试切换到该设备
- 记住用户上次选择的默认输出设备
- 支持开机启动
- 使用辅助功能权限驱动控制中心声音菜单切换输出

## 当前交互规则

- 默认输出设备：绿色文字，并在右侧显示“默认输出”
- 当前输出但不是默认输出：蓝色文字
- 既不是当前输出也不是默认输出：灰色，和图标颜色保持一致
- 鼠标悬停到非默认输出项时，右侧显示“设为默认输出”按钮
- 定时巡检不会主动展开系统声音菜单
- 只有在启动、系统音频设备变更、用户手动选择默认输出时，才会触发主动切换逻辑
- 如果输出列表读取失败，界面会直接显示失败原因，并允许手动刷新列表

## 权限要求

应用依赖 macOS 辅助功能权限访问控制中心的声音输出菜单，否则无法自动切换输出设备。

首次使用时请在下面位置启用权限：

系统设置 > 隐私与安全性 > 辅助功能

为 HomePodMonitor 打开权限后，重新回到菜单栏应用即可。

## 默认输出缓存

用户最后选择的默认输出名称会保存到 UserDefaults，键名为 preferredTargetName。

应用下次启动时会优先恢复这个值；如果本地没有缓存，则回退到默认目标名称“家庭影院”。

## 项目结构

- HomePodMonitor/HomePodMonitorApp.swift：菜单栏应用入口
- HomePodMonitor/ContentView.swift：菜单栏窗口 UI
- HomePodMonitor/AudioDeviceController.swift：音频状态监控、默认输出管理、切换流程
- HomePodMonitor/SoundOutputAccessibility.swift：辅助功能访问控制中心声音面板

## 构建

要求：

- macOS
- Xcode 17 或兼容版本

本地构建命令：

```bash
xcodebuild -project HomePodMonitor.xcodeproj -scheme HomePodMonitor -configuration Debug build
```

## 使用方式

1. 启动应用后，先授予辅助功能权限。
2. 打开菜单栏图标，查看声音菜单中的可用输出。
3. 将鼠标移到目标输出项上，点击“设为默认输出”。
4. 应用会缓存这次选择，并在后续启动时继续使用同一个目标设备。

## 实现说明

应用同时使用两条路径处理音频状态：

- CoreAudio：读取当前系统默认输出设备
- Accessibility：读取和操作控制中心声音菜单中的输出列表

其中，后台巡检只通过 CoreAudio 判断当前输出是否偏离目标，避免频繁展开系统菜单影响体验。只有需要主动切换时，才会通过辅助功能去操作系统声音菜单。

## 已知限制

- 自动切换依赖控制中心和声音菜单的可访问性结构，macOS 大版本升级后可能需要重新适配
- 设备名称匹配目前基于字符串比较，如果系统返回名称发生变化，可能导致匹配失败
- 构建时可能出现与 Swift 并发或 CoreAudio 指针相关的 warning，但当前 Debug 构建可以通过
